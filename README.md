# Audio Generation Suite (Music + Vocals + SFX)

Настольное приложение на Python с GUI (PySide6) для генерации:
- инструментальной музыки по текстовому описанию;
- полноценных музыкальных треков с вокалом;
- звуковых эффектов (SFX, атмосферные шумы и т.п.).

В основе:
- **ACE-Step 1.5** — открытый foundation‑движок, который генерирует и инструментал,
  и полные песни (музыка + вокал) из текстового описания и/или текста песни.

## 1. Окружение

Рекомендуемая конфигурация:
- **OS**: Windows 10/11 x64.
- **Python**: 3.10+.
- **GPU**: NVIDIA RTX 3050 Ti (4 ГБ VRAM) или аналогичная.

### Шаг 1. Создать виртуальное окружение

```bash
python -m venv .venv
.venv\Scripts\activate
```

### Шаг 2. Установить зависимости проекта

```bash
pip install -r requirements.txt
```

Также потребуется **PyTorch** (ACE-Step тянет его как зависимость) и **FFmpeg**:

- скачать сборку FFmpeg для Windows;
- добавить путь к `bin` в переменную окружения `PATH`.

## 2. Backend: ACE-Step 1.5

Проект использует ACE-Step 1.5 как основной движок генерации.
Пакет подтягивается из GitHub (см. `requirements.txt`), при первом запуске
модели будут автоматически скачаны и закешированы в пользовательскую папку.

На RTX 3050 Ti (4 ГБ VRAM) рекомендуется:
- запускать на GPU с включённым offload (по умолчанию в проекте),
- не держать параллельно много генераций.

Промпты поддерживаются на **50+ языках**, в том числе на русском.

### LoRA 

По умолчанию используется **базовая** модель ACE-Step без адаптеров. Можно подключить до **трёх** отдельных LoRA (инструментал, песня с вокалом, SFX), чтобы не смешивать стили в одном адаптере.

**Основной способ (без настроек окружения):** после дообучения скопируйте папку адаптера так, чтобы в каталоге репозитория лежал файл:

`loras/<режим>/pytorch_lora_weights.safetensors`

| Каталог | Режим в приложении |
|---------|---------------------|
| `loras/instrumental/` | Инструментал и вариации |
| `loras/vocal/` | Песня (музыка + вокал) |
| `loras/sfx/` | SFX / атмосферы |

Если для режима этого файла нет, для него используется базовая модель. Файлы весов в `loras/` не коммитятся в git (см. `.gitignore`).

**Дополнительно:** можно переопределить путь или указать репозиторий Hugging Face через переменные окружения (имеют приоритет над папками `loras/`): `ACE_STEP_LORA_INSTRUMENTAL`, `ACE_STEP_LORA_VOCAL`, `ACE_STEP_LORA_SFX`.

Дообучение LoRA на GPU рассчитано на **Kaggle**; в каталоге `notebooks/` лежат три отдельных Jupyter-ноутбука — по одному на каждый тип контента (`ace_step_lora_*_kaggle.ipynb`).

## 3. Запуск GUI‑приложения

```bash
python -m app.main
```

Откроется окно с вкладками:

| Вкладка | Описание |
|---------|----------|
| **Музыка (инструментал)** | Генерация инструментала по описанию; вариации на основе существующего трека; сохранение и загрузка пресетов |
| **Песня (музыка + вокал)** | Одновременная генерация музыки и вокала по тексту песни; настройки стиля, манеры, интенсивности; опция фоновых партий (backing vocals); пресеты |
| **SFX / атмосферы** | Генерация звуковых эффектов (atmosphere, short_sfx, transition, background); пресеты |
| **Проект** | Список треков, предпрослушивание (play/pause/stop, ползунок позиции), экспорт в WAV/MP3/FLAC, переименование и удаление треков, открытие папки проекта |

Слева — список проектов: создание (Музыка / Песня / SFX), переименование, удаление; выбор **тёмной** или **светлой** темы интерфейса.

## 4. Быстрый тест из CLI

Генерация инструментала без GUI (результат сохраняется в `projects_cli/`):

```bash
python -m app.cli.generate_instrumental --prompt "epic orchestral trailer, dark and tense" --duration 15
```

Полноценные песни (музыка + вокал) удобнее генерировать из GUI.

## 5. Хранение данных

- **projects/** — папка с проектами и сгенерированными треками (в корне репозитория).
- **presets/** — сохранённые пресеты параметров генерации.
- **loras/** — необязательные LoRA-адаптеры (подкаталоги `instrumental`, `vocal`, `sfx`), см. раздел «LoRA» выше.

Каталоги `projects/` и `presets/` добавлены в `.gitignore`; большие веса в `loras/**/*.safetensors` также игнорируются.
